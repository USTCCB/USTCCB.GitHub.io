🚀 Railway 后端部署指南

10 分钟完成全栈应用部署

📋 部署信息
• 前端已部署: https://frontend-pi-six-0l9p0rwmqj.vercel.app
• GitHub 仓库: USTCCB/USTCCB.GitHub.io
• 后端目录: backend
• 预计时间: 10 分钟
1 访问 Railway 并登录

1. 打开浏览器，访问 Railway 官网

打开 Railway

2. 点击右上角 Login 按钮

3. 选择 Login with GitHub

4. 授权 Railway 访问你的 GitHub 账号

2 创建新项目

1. 登录后，点击 New Project

2. 选择 Deploy from GitHub repo

3. 在列表中找到并选择: USTCCB/USTCCB.GitHub.io

4. Railway 会自动检测到这是一个 Node.js 项目

3 添加 PostgreSQL 数据库

1. 在项目页面，点击右上角 + New

2. 选择 Database

3. 选择 Add PostgreSQL

4. Railway 会自动创建数据库并生成连接信息

✅ PostgreSQL 数据库创建成功！
4 添加 Redis 缓存

1. 再次点击 + New

2. 选择 Database

3. 选择 Add Redis

✅ Redis 缓存创建成功！
5 配置后端服务

1. 点击 GitHub 仓库服务卡片（显示为 USTCCB.GitHub.io）

2. 点击 Settings 标签

3. 找到 Root Directory，输入:

backend

4. 找到 Start Command，输入:

node dist/main.js
6 配置环境变量

1. 在后端服务页面，点击 Variables 标签

2. 点击 + New Variable

3. 逐个添加以下环境变量（复制粘贴）:

⚠️ 注意：${{Postgres.PGHOST}} 这种格式是 Railway 的引用变量，会自动替换为实际值，直接复制即可！

基础配置:

NODE_ENV=production
PORT=3001

数据库配置:

DB_HOST=${{Postgres.PGHOST}}
DB_PORT=${{Postgres.PGPORT}}
DB_NAME=${{Postgres.PGDATABASE}}
DB_USER=${{Postgres.PGUSER}}
DB_PASSWORD=${{Postgres.PGPASSWORD}}

Redis 配置:

REDIS_HOST=${{Redis.REDIS_HOST}}
REDIS_PORT=${{Redis.REDIS_PORT}}

JWT 配置:

JWT_SECRET=cea67da6c7e4db6858326ba5640b62a982575c6e775d0cf2cb0719a7231ea5f1
JWT_EXPIRES_IN=7d

CORS 配置:

CORS_ORIGIN=https://frontend-pi-six-0l9p0rwmqj.vercel.app
7 部署后端

1. 所有环境变量配置完成后，点击 Deploy 按钮

2. Railway 会自动执行:

  • 拉取 GitHub 代码
  • 安装依赖 (npm install)
  • 构建 TypeScript (npm run build)
  • 启动服务 (node dist/main.js)

3. 等待部署完成（通常需要 2-5 分钟）

💡 提示：可以在 Deployments 标签查看实时日志
8 获取后端 URL

1. 部署成功后，进入 Settings 标签

2. 找到 Domains 部分

3. 点击 Generate Domain

4. Railway 会生成一个公开 URL，类似:

https://ustccbgithubio-production.up.railway.app

5. 复制这个 URL，后面会用到！

9 初始化数据库

1. 点击 PostgreSQL 服务卡片

2. 点击 Data 标签

3. 点击 Query 按钮

4. 打开本地文件: backend/src/db/schema.sql

5. 复制所有内容，粘贴到 Railway 的查询编辑器

6. 点击 Run Query

✅ 数据库表结构创建成功！
10 测试后端 API

1. 打开浏览器，访问:

https://你的Railway后端URL/health

2. 应该看到以下响应:

{
  "status": "ok",
  "timestamp": "2026-03-11T..."
}
🎉 后端部署成功！API 正常运行！
11 更新前端环境变量

1. 访问 Vercel Dashboard

2. 找到你的 frontend 项目

3. 点击 Settings

4. 点击 Environment Variables

5. 添加新变量:

Name: NEXT_PUBLIC_API_URL
Value: https://你的Railway后端URL (不要加 /api)

6. Environment: 选择 Production, Preview, Development

7. 点击 Save

12 重新部署前端

1. 在 Vercel 项目页面，点击 Deployments 标签

2. 点击最新部署右侧的 ... 菜单

3. 选择 Redeploy

4. 等待重新部署完成（约 1-2 分钟）

🎉 前端重新部署成功！现在前后端已连接！

? 常见问题

1. 后端部署失败

• 检查 Deployments 标签的日志

• 确认环境变量配置正确

• 确认 Root Directory 设置为 backend


2. 数据库连接失败

• 确保 PostgreSQL 服务正在运行

• 确保环境变量使用了 ${{Postgres.PGHOST}} 格式


3. CORS 错误

• 确保 CORS_ORIGIN 设置为前端 Vercel URL

• 不要在 URL 末尾加斜杠

• 重新部署后端


4. 前端无法连接后端

• 确保 Vercel 环境变量 NEXT_PUBLIC_API_URL 已设置

• 确保后端 URL 正确（不要加 /api）

• 重新部署前端
